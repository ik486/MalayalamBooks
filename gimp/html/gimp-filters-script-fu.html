10.6. The “Script-Fu” Submenu

ചിത്രം 16.204. The “Script-Fu” submenu


ഈ ഉപമെനുവുണ്ട് ചില സ്ക്രിപ്റ്റ്-ഫു കല്പിക്കുന്നു പ്രത്യേകിച്ച് സ്ക്രിപ്റ്റ്-ഫു കൺസോൾ അടങ്ങിയിരിക്കുന്നു. Script-Fu നിങ്ങൾ GIMP ഒരു പരമ്പര സ്വയം കല്പിക്കുന്നു പ്രവർത്തിപ്പിക്കാൻ അനുവദിക്കുന്ന സ്ക്രിപ്റ്റുകൾ, എഴുതാനുള്ള ഒരു ഭാഷയാണ്.

10.6.1. Activating the submenu

  • You can access this command from the image menu through Filters → Script-Fu

10.6.2. Refresh Scripts

നിങ്ങൾ ഈ കമാൻഡ് ചേർക്കാൻ ഓരോ തവണ വേണമെങ്കിൽ നീക്കം ചെയ്യുക അല്ലെങ്കിൽ ഒരു സ്ക്രിപ്റ്റ്-ഫു സ്ക്രിപ്റ്റ് മാറ്റും. കമാൻഡ് റീലോഡ് സ്ക്രിപ്റ്റ്-Fus അടങ്ങിയിരിക്കുന്ന മെനുകൾ ആദ്യം മുതൽ പണിയും തിരക്കഥാരചന Fus കാരണമാകുന്നു. നിങ്ങൾ വീണ്ടും തുടങ്ങുംവരെ ഈ കമാൻഡ് ഉപയോഗിക്കുന്നില്ലെങ്കിൽ, GIMP നിങ്ങളുടെ മാറ്റങ്ങൾ ശ്രദ്ധിക്കപ്പെടാതെ ചെയ്യും.

നിങ്ങൾക്ക് എന്തെങ്കിലും ഫീഡ്ബാക്ക് ലഭിക്കില്ല എന്ന് നിങ്ങളുടെ സ്ക്രിപ്റ്റുകൾ ഒന്ന് പരാജയപ്പെടുകയാണെങ്കിൽ, സംരക്ഷിക്കാതെ അല്ലാതെ, ശ്രദ്ധിക്കുക.

10.6.3. സ്ക്രിപ്റ്റ്-ഫു കൺസോൾ

സ്ക്രിപ്റ്റ്-ഫു കൺസോൾ പരസ്പരം പദ്ധതി കമാൻഡുകൾ പരീക്ഷിക്കാൻ കഴിയുന്ന ഒരു ഡയലോഗ് വിൻഡോ ആണ്.

കൺസോൾ ഔട്ട്പുട്ട് പദ്ധതി കമാൻഡുകൾ ടൈപ്പ് ഉപയോഗിക്കുന്ന ഒരു ടെക്സ്റ്റ്ബോക്സ് ഒരു വലിയ സ്ക്രോൾ പ്രധാന ജാലകം ഉൾക്കൊള്ളുന്നു. നിങ്ങൾ ഒരു പദ്ധതി പ്രസ്താവന ടൈപ്പുചെയ്യുക തുടർന്ന് Enter കീ അമർത്തുമ്പോൾ, കമാൻഡ് അതിന്റെ മടക്കം മൂല്യം പ്രധാന വിൻഡോയിൽ ദൃശ്യമാകും.

ചിത്രം 16.205. The Script-Fu Console


നിങ്ങൾ Scheme ഉദാഹരണങ്ങൾ എത്ര വിഭാഗം 3, “ഒരു സ്ക്രിപ്റ്റ്-ഫു ട്യൂട്ടോറിയൽ” ലെ Script-Fu console ഉപയോഗിക്കാൻ കുറിച്ച് കൂടുതൽ വിവരങ്ങൾ കണ്ടെത്തും.

The Script-Fu Console Buttons

Browse

ഈ ബട്ടൺ പദ്ധതി ടെക്സ്റ്റ്ബോക്സ് കല്പിക്കുന്നു അടുത്താണ്. ക്ലിക്കുചെയ്യുമ്പോൾ, procedure browser വിൻഡോയുടെ താഴെയുള്ള അധിക ബട്ടൺ ഉപയോഗിച്ച്, മാറികിട്ടി:

The additional button of the Procedure Browser

നടപടി ബ്രൗസറിൽ ഈ Apply ബട്ടൺ അമർത്തുമ്പോൾ, തിരഞ്ഞെടുത്ത നടപടിക്രമം ടെക്സ്റ്റ് ബോക്സിൽ ഒട്ടിക്കുകയുള്ളൂ:

അപ്ലൈഡ് PDB നടപടിക്രമം

ഇപ്പോൾ നിങ്ങൾക്ക് വെറും (ഇവിടെ: “layer” ആൻഡ് “mode”) പാരാമീറ്റർ പേരുകൾ മാറ്റിസ്ഥാപിക്കാൻ ഞങ്ങൾക്കുണ്ട് യഥാർത്ഥ മൂല്യങ്ങൾ കൂടെ, തുടർന്ന് Enter അമർത്തി നടപടിക്രമം വിളിക്കാം.

Close

ഈ ബട്ടൺ അമർത്തിയാൽ സ്ക്രിപ്റ്റ്-ഫു കൺസോൾ അടയ്ക്കുന്നു.

തെളിഞ്ഞ

Wenn ഈ ബട്ടൺ ക്ലിക്ക് പ്രധാന വിൻഡോയുടെ ഉള്ളടക്കം നീക്കംചെയ്യും. നിങ്ങൾ Save കമാൻഡ് ഉപയോഗിച്ച് നീക്കംചെയ്ത ഉള്ളടക്കങ്ങൾ തിരികെ ലഭിക്കില്ല എന്ന് ശ്രദ്ധിക്കുക.

രക്ഷിക്കും

ഈ കമാൻഡ് പ്രധാന ജാലകത്തിന്റെ ഉള്ളടക്കം സംരക്ഷിക്കാൻ ആ സ്ക്രിപ്റ്റ്-ഫു കൺസോൾ ഔട്ട്പുട്ട് (- പ്രതീകങ്ങൾ “>” ഉൾപ്പെടെ) ആണ് അനുവദിക്കുന്നു.

10.6.4. Start Server

ഈ കമാൻഡ് വായിച്ചു ഒരു പ്രത്യക പോർട്ട് വഴി അവനെ അയക്കും സ്ക്രിപ്റ്റ്-ഫു (സ്കീം) പ്രസ്താവനകൾ നടപ്പിലാക്കുന്നു സെർവറിൽ, ആരംഭിക്കും.

ചിത്രം 16.206. The Script-Fu Server Options


Server Port

സ്ക്രിപ്റ്റ്-ഫു സെർവർ ശ്രദ്ധിക്കും എവിടെ പോർട്ട് നമ്പർ. വ്യത്യസ്ത പോർട്ട് നമ്പറുകൾ, തീർച്ചയായും വ്യക്തമാക്കിക്കൊണ്ട്, ഒന്നിലധികം സെർവർ ആരംഭിക്കാൻ സാധ്യമാണ്.

Server Logfile

ഓപ്ഷണൽ ഒരു ഫയൽ സെർവർ അനൌപചാരികശൈലിയും പിശക് സന്ദേശങ്ങൾ ലോഗിൻ ചെയ്യാൻ ഉപയോഗിക്കുന്ന പേര് വ്യക്തമാക്കാൻ കഴിയും. യാതൊരു ഫയൽ വ്യക്തമാക്കിയിട്ടുണ്ടെങ്കിൽ, സന്ദേശങ്ങൾ STDOUT എന്നതിലേക്ക് എഴുതുന്നു.

The Script-Fu Server Protocol

സ്ക്രിപ്റ്റ്-ഫു സെർവറുമായി ആശയവിനിമയം ഉപയോഗിച്ച് പ്രോട്ടോക്കോൾ വളരെ എളുപ്പമാണ്:

  • സെർവറിലേക്ക് അയച്ചു നീളം L ഓരോ സന്ദേശം (സ്ക്രിപ്റ്റ്-ഫു പ്രസ്താവന) താഴെ 3 ബൈറ്റുകൾ ഉപയോഗിച്ച് പിൻതുടരുന്നതായിരിക്കണം ഉണ്ട്:

    പട്ടിക 16.1. കമാൻഡുകൾ തലക്കെട്ടിന് ഫോർമാറ്റ്

    Byte # Content വിവരണം
    0 0x47 മാജിക് ബൈറ്റ് ( 'ജി')
    1 എൽ DIV 256 L ഹൈ ബൈറ്റ്
    2 എൽ സിനിമ 256 L കുറഞ്ഞ ബൈറ്റ്

  • നീളം L എന്ന സെർവർ (മടക്കം മൂല്യം അല്ലെങ്കിൽ പിശക് സന്ദേശം) മുതൽ ഓരോ പ്രതികരണവും താഴെ 4 ബൈറ്റുകൾ കൂടെ പിൻതുടരുന്നതായിരിക്കണം ചെയ്യും:

    പട്ടിക 16.2. പ്രതികരണങ്ങൾക്കായി ഹെഡ്ഡർ ഫോർമാറ്റ്

    Byte # Content വിവരണം
    0 0x47 മാജിക് ബൈറ്റ് ( 'ജി')
    1 error code വിജയത്തെ 0, 1 പിശക് ന്
    2 എൽ DIV 256 L ഹൈ ബൈറ്റ്
    3 എൽ സിനിമ 256 L കുറഞ്ഞ ബൈറ്റ്

സൂചന

നിങ്ങളുടെ കൈകൾ വൃത്തികെട്ട നേടുകയും വേണ്ടെങ്കിൽ: servertest.py പേരുള്ള ഒരു പൈത്തൺ സ്ക്രിപ്റ്റ് നിങ്ങൾക്ക് സ്ക്രിപ്റ്റ്-ഫു സെർവർ ഒരു ലളിതമായ കമാൻഡ് ലൈൻ ഷെൽ ആയി ഉപയോഗിക്കാനും കഴിയുന്ന GIMP സോഴ്സ് കോഡ്, അവിടെ ഷിപ്പുചെയ്തു.
