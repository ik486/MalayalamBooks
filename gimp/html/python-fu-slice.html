14.4. പൂളുക

14.4.1. പൊതു അവലോകനം

ചിത്രം 17.351. “Slice” ഫിൽട്ടർ ഉദാഹരണം

ഗൈഡുകൾ യഥാർത്ഥ ചിത്രം

“Slice” പ്രയോഗിച്ചു


ഈ ഫിൽട്ടർ ഒരു ലളിതമായ HTML ഫയലുകൾ ഉപയോഗിക്കുന്ന സെൻസിറ്റീവ് ചിത്രങ്ങൾ സൃഷ്ടിക്കാൻ സഹായി ഉപയോഗിക്കാൻ എളുപ്പമാണ്. ഫിൽട്ടർ കഷണങ്ങൾ അപ്പ് സോഴ്സ് ചിത്രം (Guillotine കമാൻഡ് പോലെ ൻ) അതിന്റെ തിരശ്ചീന, ലംബ ഗൈഡുകൾ സഹിതം, ഉപ-ഒരു കൂട്ടം ചിത്രങ്ങൾ ഉത്പാദിപ്പിക്കുന്നത്. ഇതേസമയം ഇത് ഒരു ടെക്സ്റ്റ് ഫയലിൽ സംരക്ഷിച്ചു ഒരു മേശ HTML കോഡ് ഒരു കഷണം സൃഷ്ടിക്കുന്നു. ഓരോ പട്ടിക സെൽ ചിത്രം ഒരു ഭാഗത്ത് അടങ്ങിയിരിക്കുന്നു. ടെക്സ്റ്റ് ഫയൽ പിന്നീട് ഒരു HTML പ്രമാണം ഉൾച്ചേർക്കണോ.

ഈ ഫിൽട്ടർ ശരിക്കും വളരെ ലളിതമായ തുണ ശ്രദ്ധിക്കുക. അരിപ്പ നിർമ്മിക്കുന്ന ഒരു സാധാരണ HTML കോഡ് അതിനെക്കാൾ എത്ര അധികം ആയിരിക്കാം:

ഉദാഹരണം 17.1. ലളിതമായ “Slice” ഫിൽട്ടർ ഉദാഹരണം ഔട്ട്പുട്ട്

ചിത്രം യാതൊരു ഗൈഡുകൾ അവിടെ തന്നെ അരിപ്പ ഒന്നും NO ചെയ്യും. എന്നാൽ, വഴികാട്ടികൾ വെറും മറച്ചിരിക്കുന്നു, അരിപ്പ പ്രവർത്തിക്കും.

14.4.3. ഓപ്ഷനുകൾ

ചിത്രം 17.352. “Slice” ഓപ്ഷനുകൾ


ഏറ്റവും ഓപ്ഷനുകൾ സ്വയം വിശദീകരിക്കുന്ന, എന്നാൽ എങ്കിലും:

എച്ച്ടിഎംഎൽ കയറ്റുമതി പാത

എവിടെ HTML ഫയലിൽ ചിത്ര ഫയലുകളിൽ സംരക്ഷിക്കും. സ്വതവേ ഈ ഫയലുകൾ ഇപ്പോൾ പ്രവർത്തിക്കുന്ന ഡയറക്ടറിയിൽ സംഭരിക്കും. വലതുവശത്ത് ബട്ടൺ ക്ലിക്കുചെയ്താൽ മറ്റൊരു ലൊക്കേഷൻ തിരഞ്ഞെടുക്കുക കഴിയുന്ന ഒരു പുൾ-ഡൌൺ മെനു തുറക്കുന്നു.

കയറ്റുമതി ഫയലിന്റെ

HTML ഫയലിൽ പേര്. നിങ്ങൾ ടെക്സ്റ്റ്ബോക്സ് ഉപയോഗിച്ച് ഫയൽനാമം മാറ്റാൻ കഴിയും.

ചിത്രത്തിന്റെ പേര് പ്രിഫിക്സ്

slice: ഈ ഫിൽട്ടർ നിർമ്മിക്കുന്ന ഒരു ഇമേജ് ഫയലിന്റെ പേര് prefix_i_k. ext, prefix നിങ്ങൾ സ്വതന്ത്രമായി സ്വതവേ, വലതുവശത്ത് ടെക്സ്റ്റ്ബോക്സ് ഉപയോഗിച്ച് തെരഞ്ഞെടുക്കാൻ സാധ്യമാകുന്ന ഹലോ ആ ഭാഗമാണ് എവിടെ. (i ആൻഡ് k വരി നമ്പറുകളും കോളം, ഓരോ 0 തുടങ്ങുന്ന ആകുന്നു; .ext തിരഞ്ഞെടുത്ത Image format അനുസരിച്ച് എക്റ്റെൻഷനുള്ള ആണ്.)

നിങ്ങൾ Javascript for onmouseover and clicked സൃഷ്ടിക്കുന്നതിനും ചിത്രങ്ങൾ വിവിധ സെറ്റുകൾ ആവശ്യമാണ് ഈ ഓപ്ഷൻ ഉപയോഗപ്പെടുന്നു.

ചിത്രം ഫോർമാറ്റ്

നിങ്ങൾ GIF, JPG, അല്ലെങ്കിൽ PNG file format ഇമേജ് ഫയലുകൾ സൃഷ്ടിക്കാൻ തിരഞ്ഞെടുക്കാം.

പ്രത്യേക ഇമേജ് ഫോൾഡർ, ചിത്രം കയറ്റുമതി ഫോൾഡർ

Separate image folder പ്രാപ്തമാക്കി, ചിത്രം ഫയലുകൾ സ്ഥാപിക്കപ്പെടും അവിടെ ഒരു ഫോൾഡർ സൃഷ്ടിക്കപ്പെടും. സ്വതവേ, ഈ ഡെസ്റ്റിനേഷൻ ഫോൾഡർ പേര് images, നിങ്ങളിൽ Folder for image export ടെക്സ്റ്റ്ബോക്സ് ഇത് മാറ്റാം.

ഉദാഹരണം 17.2. പ്രത്യേക ഇമേജ് ഫോൾഡറിൽ

പ്രാപ്തമാക്കിയ “Separate image folder” ഫലം


മേശ ഘടകങ്ങൾ തമ്മിലുള്ള സ്പെയ്സ്

ഈ വില (0-15) HTML പട്ടിക ലേക്ക് “cellspacing” ആട്രിബ്യൂട്ട് പാസുചെയ്യും. ഫലം തിരശ്ചീന, ലംബ വഴികാട്ടികളേ വ്യക്തമാക്കിയ വീതി അടി മാറ്റിസ്ഥാപിക്കും എന്നു ആകുന്നു

ഉദാഹരണം 17.3. മേശ ഘടകങ്ങൾ തമ്മിലുള്ള സ്പെയ്സ്

പൊരുത്തപ്പെടുന്ന HTML കോഡ് സ്നിപ്പറ്റ്


ചിത്രം ഈ അടിക്കു വലിപ്പം വലുതായ ചെയ്യില്ല ശ്രദ്ധിക്കുക. പകരം, ഇറേസർ ടൂൾ ഉപയോഗിച്ച് തല്ലും വരച്ചു പോലെ ഫലമായുണ്ടാകുന്ന എച്ച്ടിഎംഎൽ ചിത്രം കാണപ്പെടും.

onmouseover ജാവാ സ്ക്രിപ്റ്റ് ക്ലിക്കുചെയ്തെന്നുമുള്ളവ

ഈ പ്രയോഗം പ്രവർത്തന, അരിപ്പ ചില JavaScript കോഡ് ചേർക്കും. HTML കോഡ് തോന്നുന്നു, ഈ കോഡ് മറിച്ച് ചില ചലനാത്മക പ്രവർത്തനക്ഷമത ചേർത്ത് ഒരു നല്ല ആരംഭസ്ഥാനം, അതേ പടി പ്രവർത്തിക്കുന്നില്ല. JavaScript കോഡ് “onmouseover” പോലുള്ള ഇവന്റുകൾ കൈകാര്യം ഒരു ചടങ്ങിൽ ലഭ്യമാക്കുന്നു:

ഉദാഹരണം 17.4. JavaScript കോഡ് സ്നിപ്പെറ്റ് 

function exchange (image, images_array_name, event)
  {
    name = image.name;
    images = eval (images_array_name);

    switch (event)
      {
        case 0:
          image.src = images[name + "_plain"].src;
          break;
        case 1:
          image.src = images[name + "_hover"].src;
          break;
        case 2:
          image.src = images[name + "_clicked"].src;
          break;
        case 3:
          image.src = images[name + "_hover"].src;
          break;
      }
  }

മേശ ക്യാപ്സ് വേണ്ടി ആനിമേഷൻ ഒഴിവാക്കുക

പ്രവർത്തനരഹിതമാക്കുമ്പോൾ, അരിപ്പ ഓരോ പട്ടിക സെൽ ഒരു <a href="# "> ... </a> ഹൈപ്പർലിങ്ക് സ്റ്റബ് ചേർക്കും. പ്രാപ്തമായിരിക്കുമ്പോൾ (ഈ സ്വതവേ), കുറഞ്ഞത് രണ്ടു ലംബമായ തിരശ്ചീനമായി രണ്ടു ഗൈഡുകൾ ഉണ്ട്, അരിപ്പ ഒരു നിര അല്ലെങ്കിൽ വരി ആദ്യ, അവസാന സെൽ ഹൈപ്പർലിങ്ക് അപൂർണ്ണമാണ് ചേർക്കില്ല. നിങ്ങൾ അതിർനാടും ചിത്രം ഉണ്ടെങ്കിൽ നിങ്ങൾ അതിർത്തി സെൻസിറ്റീവ് വരുത്തുവാൻ ആവശ്യമില്ലാത്തപ്പോൾ ഈ ക്രമീകരണം സഹായിക്കും.
