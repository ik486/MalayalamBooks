3.2. വേര്യബിൾ പ്രവർത്തനങ്ങൾ

ഇപ്പോൾ ഞങ്ങൾ ഓരോ പദ്ധതി പ്രസ്താവന ബ്രാക്കറ്റിൽ ചുറ്റപ്പെട്ടു ഫങ്ഷൻ പേര് / ഓപ്പറേറ്റർ ആദ്യം ലിസ്റ്റ് ചെയ്തിട്ടുണ്ടോയെന്ന് അറിയുന്നു എന്നു ഞങ്ങൾ സൃഷ്ടിക്കാനും എങ്ങനെ ഉപയോഗിക്കണമെന്ന് വേരിയബിളുകൾ അറിഞ്ഞിരിക്കണം, എങ്ങനെ സൃഷ്ടിക്കാനും ഫംഗ്ഷനുകളും ഉപയോഗിക്കാൻ. നമുക്ക് വേരിയബിളുകള്ക്കായി കൊണ്ട് ആരംഭിക്കും.

3.2.1. പ്രസ്താവിക്കുന്നു വേരിയബിള്സ്

വിവിധ രീതികൾ ഒരു ദമ്പതികൾ വേരിയബിളുകൾ നിർവചിക്കാൻ ഉണ്ടെങ്കിലും രീതി let* പരവലയങ്ങൾക്കു് ഉപയോഗിക്കുക എന്നതാണ്. മറ്റ് പ്രോഗ്രാമിങ് ഭാഷകളിൽ മുൻപരിചയം ഉണ്ടെങ്കിൽ, ഈ പരവലയങ്ങൾക്കു് ലോക്കൽ വേരിയബിളുകൾ അന്ന് അവർ സജീവമായി മുൻപരിചയം ഒരു സ്കോപ്പ് ഒരു ലിസ്റ്റ് നിർവ്വചിക്കുമ്പോൾ തുല്യമാണ്. ഉദാഹരണമായി, യഥാക്രമം രണ്ടു ചരങ്ങളുടെ, എ, ബി 1, 2 സമാരംഭിച്ചിട്ടില്ല, പ്രഖ്യാപിക്കാൻ നിങ്ങൾ എഴുതുന്ന ആഗ്രഹിക്കുന്നു:

        (let*
           (
              (a 1)
              (b 2)
           )
           (+ a b)
        )

അല്ലെങ്കിൽ, ഒരു വരിയിൽ ആയി:

(* ചെയ്യട്ടെ ((1) (ബി 2)) (+ ഒരു ബി))
കുറിപ്പു്

നിങ്ങൾ കൺസോൾ വിൻഡോ ഉപയോഗിക്കുകയാണെങ്കിൽ ഇതെല്ലാം ഒരു ലൈനിൽ ഇട്ടു വരും. പൊതുവായി പറഞ്ഞാൽ, നിങ്ങളുടെ സ്ക്രിപ്റ്റുകൾ കൂടുതൽ സൗകര്യമാകും സഹായിക്കുന്നതിന് ഇൻഡെൻറേഷൻ സമാനമായ വഴിക്കു ആഗ്രഹമുണ്ടാകും. നാം വൈറ്റ് സ്പേസ് വിഭാഗത്തിൽ ഈ കുറച്ച് സംസാരിക്കാം കാണാം.

ഈ രണ്ടു ലോക്കൽ വേരിയബിളുകൾ, എ, ബി പ്രഖ്യാപിക്കുന്നു അവരെ അതിനു് പിന്നീട് രണ്ടു ചരങ്ങളുടെ ആകെത്തുകയാണ് പ്രിന്റ് ചെയ്യും.

3.2.2. ഒരു പ്രാദേശിക വേരിയബിൾ എന്താണ്?

ഞങ്ങൾ let* ആവിഷ്കാര പരാന്തിസിസുകൾ ഉള്ളിൽ, അത് ശേഷം സമ്മേഷന്റെ (+ a b) എഴുതി നിങ്ങൾ ശ്രദ്ധിക്കണം.

let* പ്രസ്താവന ഏത് ഡിക്ലയർ വേരിയബിളുകൾക്ക് കഴിയി നിങ്ങളുടെ സ്ക്രിപ്റ്റിൽ ഒരു പ്രദേശത്തെ നിഷ്കർഷിക്കുന്നു ഇതിൻറെ കാരണം; നിങ്ങൾ (let* ) പ്രസ്താവനയ്ക്ക് ശേഷം (+ a b) പ്രസ്താവന ടൈപ്പ് ചെയ്താൽ, നിങ്ങൾ പ്രഖ്യാപിച്ചു വേരിയബിളുകൾ let* പ്രസ്താവന പശ്ചാത്തലത്തിൽ മാത്രം സാധുവായ കാരണം ഒരു പിശക് ലഭിക്കും; അവർ പ്രോഗ്രാമർമാരുടെ ലോക്കൽ വേരിയബിളുകൾ വിളിക്കുന്ന ആകുന്നു.

3.2.3. ജനറൽ സിന്റാക്സ് let* ഓഫ്

ഒരു let* പ്രസ്താവന ജനറൽ ഫോമാണ്:

        (let* ( variables )
          expressions )

എവിടെ വേരിയബിളുകൾ ഉദാ പരാന്തിസിസുകൾ ഉള്ളിൽ പ്രഖ്യാപിക്കുന്നത്, (a 2), എക്സപ്രഷനുകളും സാധുവായ പദ്ധതി പദപ്രയോഗങ്ങൾ ഉണ്ട്. ഇവിടെ പ്രഖ്യാപിച്ചു വേരിയബിളുകൾ let* പ്രസ്താവന മാത്രം സാധുവായ ആകുന്നു ഓർക്കുക - അവർ ലോക്കൽ വേരിയബിളുകൾ മുൻപരിചയം.

3.2.4. വൈറ്റ് സ്പേസ്

മുമ്പ്, ഞങ്ങൾ ഒരുപക്ഷേ വ്യക്തമാക്കാം നിങ്ങളുടെ സ്ക്രിപ്റ്റുകൾ ഓർഗനൈസ് സഹായിക്കുന്നതിന് ഇൻഡെൻറേഷൻ ഉപയോഗിക്കാൻ ആഗ്രഹിക്കുന്ന കാണാം വസ്തുത പരാമർശിച്ചു. ഈ ദത്തെടുക്കാൻ ഒരു നല്ല നയം ആണ്, സ്കീം ഒരു പ്രശ്നം അല്ല - വൈറ്റ് സ്പേസ് സ്കീം ദ്വിഭാഷിയായിരുന്ന അവഗണിക്കുകയും, അതുകൊണ്ട് ഉദാരമായി ഒരു സ്ക്രിപ്റ്റ് ഉള്ളിൽ വ്യക്തമാക്കാം കോഡ് ഓർഗനൈസ് സഹായിക്കുന്നതിന് പ്രയോഗിക്കാൻ കഴിയും. എന്നിരുന്നാലും, നിങ്ങൾ Script- ഫു ന്റെ കൺസോൾ ജാലകത്തിൽ പ്രവർത്തിക്കുന്നത് എങ്കിൽ, നിങ്ങൾ ഒരു ലൈനിൽ ഒരു മുഴുവൻ പദപ്രയോഗം നൽകേണ്ടതുണ്ട്; എക്സ്പ്രഷൻ ഉദ്ഘാടന ക്ലോസിങ്ങ് പരാന്തിസിസുകൾ തമ്മിലുള്ള എല്ലാം സ്ക്രിപ്റ്റ്-ഫു കൺസോൾ വിൻഡോയിൽ ഒരു ലൈനിൽ വരണം ആണ്.

3.2.5. ഒരു ചര ഒരു പുതിയ റ്റി.എം

നിങ്ങൾ ഒരു വേരിയബിൾ സമാരംഭിച്ചിട്ടില്ല കഴിഞ്ഞാൽ നിങ്ങൾക്ക് സ്ക്രിപ്റ്റ് പിന്നീട് അതിന്റെ മൂല്യം മാറ്റാൻ ചെയ്യേണ്ടതുണ്ടു്. വേരിയബിൾ മൂല്യം മാറ്റാൻ set! പ്രസ്താവന ഉപയോഗിക്കുക:

        (let* ( (theNum 10) ) (set! theNum (+ theNum theNum)) )

മുന്നോട്ട് പോയി സ്ക്രിപ്റ്റ്-ഫു കൺസോൾ വിൻഡോയിൽ അത് നൽകുക, തുടർന്ന് മുകളിലുള്ള പ്രസ്താവന എന്തു ചെയ്യും ഊഹിക്കാൻ ശ്രമിക്കുക.

കുറിപ്പു്

“\” യാതൊരു വരി ഇല്ല സൂചിപ്പിക്കുന്നു. അത് അവഗണിക്കുക (നിങ്ങളുടെ സ്ക്രിപ്റ്റ്-ഫു കൺസോളിൽ ടൈപ്പ് ചെയ്ത് Enter ഹിറ്റ് ചെയ്യരുത് ചെയ്യരുത്) അടുത്ത വരിയിൽ തുടരുക.

3.2.6. പ്രവർത്തനങ്ങൾ

ഇപ്പോൾ നിങ്ങൾ വേരിയബിളുകളിലൊന്ന് ഹാങ് ലഭിച്ചു ന്റെ ചില പ്രവർത്തനങ്ങൾ പ്രവർത്തിക്കാൻ അനുവദിക്കുക. ഇനിപ്പറയുന്ന വാക്യഘടനയ്ക്കൊപ്പം ചടങ്ങിൽ പ്രസ്താവിക്കും:

        (define
           (
              name
              param-list
           )
           expressions
        )

name ഈ ഫംഗ്ഷൻ നിയോഗിച്ചിട്ടുള്ള പേര് എവിടെ, param-list പാരാമീറ്റർ പേരുകളുടെ സ്പേസ്-ഡിലിമിറ്റഡ് ലിസ്റ്റാണ്, expressions അതു വിളിച്ച് വരുമ്പോൾ ചടങ്ങിൽ വധശിക്ഷയ്ക്ക് പദപ്രയോഗങ്ങൾ പരമ്പരയാണ്. ഉദാഹരണത്തിന്:

(AddXY വിവിധ INX വിവരിക്കുക () (+ മറ്റൊരു INX))

AddXY ഫങ്ഷൻ പേര് ആണ് inX ആൻഡ് inY വേരിയബിളുകൾ ആകുന്നു. ഈ ഫങ്ഷൻ അതിന്റെ രണ്ട് മാനദണ്ഡങ്ങൾ എടുക്കുന്നു അവർ തമ്മിൽ ചേർക്കുന്നു.

നിങ്ങൾ (സി / സി ++, ജാവ, പാസ്കൽ തുടങ്ങിയവ പോലുള്ള) മറ്റ് ഒഴിച്ചുകൂടാനാവാത്തതാണ് ഭാഷകളിൽ പ്രോഗ്രാം ചെയ്തെങ്കിൽ, ശ്രദ്ധിച്ചിരിക്കാനിടയുണ്ട് മറ്റ് പ്രോഗ്രാമിങ് ഭാഷകളിൽ താരതമ്യപ്പെടുത്തുമ്പോൾ കാര്യങ്ങൾ ഒരു ദമ്പതികൾ ഈ ഫംഗ്ഷൻ നിർവചനത്തിൽ പരദേശികൾ ആയിരിക്കുന്നു എന്നു.

  • ആദ്യം, പരാമീറ്ററുകൾ ഏതെങ്കിലും “types” ഇല്ല അറിയിപ്പ് (എന്നതാണ് നാം അവരെ സ്ട്രിങ്ങുകൾ, അല്ലെങ്കിൽ പൂർണ്ണസംഖ്യകളുടെ മുതലായവ പോലെ വിവരിക്കും ചെയ്തില്ല). സ്കീം ഒരു type- കുറവ് ഭാഷയാണ്. ഈ ഹാൻഡി ആണ് വേഗത്തിൽ സ്ക്രിപ്റ്റ് എഴുത്തു അനുവദിക്കുന്നു.

  • രണ്ടാമതായി, നമ്മുടെ ചടങ്ങിൽ ഫലമാണ് “return” എങ്ങനെ വിഷമിക്കേണ്ടതില്ല ആവശ്യമില്ലെന്ന് അറിയിപ്പ് - ഈ ഫംഗ്ഷൻ വിളിക്കുമ്പോൾ അവസാന പ്രസ്താവന മൂല്യം “returned” ആണ്. പിന്നെ, കൺസോൾ കടന്നു ചടങ്ങിൽ ടൈപ്പ് പോലെ എന്തെങ്കിലും ശ്രമിക്കുക:

    (AddXY (AddXY 5 6) 4)
