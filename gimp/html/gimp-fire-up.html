1. ജിമ്പ് പ്രവർത്തിയ്ക്കുന്നു

പലപ്പോഴും നിങ്ങൾ (നിങ്ങളുടെ സിസ്റ്റം ഒറ്റ നിങ്ങൾക്ക് നൽകാൻ സജ്ജീകരിച്ചു എങ്കിൽ) ഒരു ഐക്കണിൽ ക്ലിക്കുചെയ്ത്, അല്ലെങ്കിൽ ഒരു കമാൻഡ് ലൈനിൽ gimp ടൈപ്പുചെയ്ത് GIMP ഒന്നുകിൽ ആരംഭിക്കുക. നിങ്ങൾ ഇൻസ്റ്റാൾ GIMP ഒന്നിലധികം പതിപ്പുകൾ ഉണ്ടെങ്കിൽ, പുതിയ പതിപ്പ് ലഭിക്കാൻ gimp-2.8 ടൈപ്പ് വരാം. നിങ്ങൾ, നിങ്ങൾ ആഗ്രഹിക്കുന്ന പക്ഷം പ്രോഗ്രാം പേര് ശേഷം കമാൻഡ് ലൈനിൽ ഇമേജ് ഫയലുകളുടെ ഒരു പട്ടിക നൽകാൻ കഴിയും, അതു ആരംഭിച്ച് പോലെ അവർ സ്വയം GIMP വഴി തുറക്കപ്പെടും. ഇത് സാധ്യമാണ് എന്നാൽ, പ്രവർത്തിച്ചുകൊണ്ടിരിക്കുന്നതിനാൽ ഒരിക്കൽ GIMP നിന്ന് ഫയലുകൾ തുറക്കാൻ ആണ്.

മിക്ക ഓപ്പറേറ്റിങ് സിസ്റ്റങ്ങൾ ബന്ധപ്പെട്ട ആപ്ലിക്കേഷൻ (അത്തരം GIMP പോലുള്ള) ഫയലുകളുടെ ഒരു ക്ലാസ് (പോലുള്ള .jpg, അവരുടെ എക്റ്റെൻഷനുള്ള നിർണ്ണയിച്ച) സഹകരിപ്പിച്ച് ഏത് ഫയൽ അസോസിയേഷനുകളുടെ, പിന്തുണയ്ക്കുന്നു. ഇമേജ് ഫയലുകൾ ശരിയായി “associated” GIMP ലേക്ക് വരുമ്പോൾ, നിങ്ങൾ GIMP ഇത് തുറക്കാൻ ഒരു ഇമേജ് ക്ലിക്ക് ഇരട്ട.

1.1. അറിയപ്പെടുന്ന പ്ലാറ്റ്ഫോമുകൾ

GIMP ഇന്ന് ലഭ്യമാണ് ഏറ്റവും ജനശ്രദ്ധ ചിത്രങ്ങൾ വരക്കുവാനുള്ള ആണ്. ഏത് GIMP പണി അറിയപ്പെടുന്നു പ്ലാറ്റ്ഫോമുകൾ ഉൾപ്പെടുന്നു:

GNU/Linux, Apple Mac OS X, Microsoft Windows, OpenBSD, NetBSD, FreeBSD, Solaris, SunOS, AIX, HP-UX, Tru64, Digital UNIX, OSF/1, IRIX, OS/2, ഒപ്പം BeOS.

GIMP കാരണം അതിന്റെ സോഴ്സ് കോഡ് ലഭ്യത മറ്റ് ഓപ്പറേറ്റിങ് സിസ്റ്റങ്ങളിൽ ലേക്ക് എളുപ്പത്തിൽ ഒന്നിൽ ആണ്. കൂടുതൽ വിവരങ്ങൾക്ക് GIMP ഡെവലപ്പർമാർ ഹോംപേജ് സന്ദർശിക്കുക. ???.

1.2. ഭാഷ

GIMP ഓട്ടോമാറ്റിക്കായി കണ്ടുപിടിച്ചു് സിസ്റ്റം ഭാഷ ഉപയോഗിക്കുന്നു. Edit → Preferences → Interface: ഭാഷ തിരിച്ചറിയൽ പരാജയപ്പെടുകയാണെങ്കിൽ, അല്ലെങ്കിൽ GIMP -2,8 ശേഷം, മറ്റൊരു ഭാഷ ഉപയോഗിക്കാൻ ആഗ്രഹിക്കുന്നു എങ്കിൽ നിങ്ങൾ നിമിത്തം അങ്ങനെ ചെയ്യാൻ കഴിയുന്ന ഇടയില്ല ഇവന്റിൽ.

നിങ്ങൾക്ക് ഉപയോഗിക്കാം:

ലിനക്സ് കീഴിൽ

In LINUX: കൺസോൾ മോഡ് ടൈപ്പ് LANGUAGE=en gimp അല്ലെങ്കിൽ LANG=en gimp ഫാ: ml പകരക്കാരനായിട്ടാണ് ദേ, ... നിങ്ങൾ ആഗ്രഹിക്കുന്ന ഭാഷയിൽ പറയുന്നെങ്കിലും. പശ്ചാത്തലം: LANGUAGE=en ഉപയോഗിക്കുന്നു വധിക്കപ്പെട്ട പ്രോഗ്രാം gimp ഒരു പരിസര വേരിയബിൾ സജ്ജമാക്കുന്നു.

വിൻഡോസ് എക്സ്പി കീഴിൽ

“System Variables” പ്രദേശത്ത് Control Panel → System → Advanced → Environment ബട്ടൺ: Add ബട്ടൺ: മൂല്ല്യത്തിനുള്ള ... പേര് ഫാ അല്ലെങ്കിൽ ദേ വേണ്ടി LANG നൽകുക. ശ്രദ്ധിക്കുക! താങ്കൾക്ക് താങ്കളുടെ ഇച്ഛാനുസരണം മൂല്യനിർണ്ണയം മൂന്നു തുടർച്ചയായി OK ക്ലിക്ക് ചെയ്യുക.

If you change languages often, you can create a batch file to change the language. Open NotePad. Type the following commands (for french for instance):

set lang=fr
start gimp-2.8.exe

Save this file as GIMP-FR.BAT (or another name, but always with a .BAT extension). Create a shortcut and drag it to your desktop.

മറ്റൊരു സാധ്യത: Start → Programs → GTK Runtime Environment അപ്പോൾ Select language നിങ്ങൾ ഡ്രോപ്പ്-ഡൌൺ പട്ടികയിൽ ഭാഷ തിരഞ്ഞെടുക്കുക.

ആപ്പിൾ മാക് ഒഎസ് എക്സ് കീഴിൽ

സിസ്റ്റം മുൻഗണനകൾ നിന്നും, അന്താരാഷ്ട്ര ഐക്കണിൽ ക്ലിക്ക് ചെയ്യുക. ഭാഷ ടാബിൽ, താൽപ്പര്യപ്പെടുന്ന പട്ടികയിലുള്ള ആദ്യത്തെ ആയിരിക്കണം.

മറ്റൊരു ജിമ്പ് ഇൻസ്റ്റൻസ്

ജിമ് ഒന്നിലധികം സംഭവങ്ങൾ പ്രവർത്തിപ്പിക്കാൻ -n ഉപയോഗിക്കുക. ഉദാഹരണത്തിന്, ഇംഗ്ലീഷിൽ GIMP മറ്റൊരു ഉദാഹരണത്തിന് ആരംഭിക്കാൻ സിസ്റ്റം ഭാഷ, LANGUAGE=en gimp-2.8 -n ലെ ജിമ്പ് ആരംഭിക്കാൻ gimp-2.8 ഉപയോഗിക്കുക; ഈ പരിഭാഷകർ വളരെ ഉപയോഗപ്പെടുന്നു.

1.3. കമാൻഡ് ലൈൻ ആർഗ്യുമെന്റുകൾ

GIMP ആരംഭിക്കുമ്പോൾ വാദങ്ങൾ ആവശ്യമില്ലാത്തവ ആണെങ്കിലും, ഏറ്റവും സാധാരണമായ വാദങ്ങൾ താഴെ കൊടുത്തിരിക്കുന്നു. ഒരു യുണിക്സ് സിസ്റ്റം ന് നിങ്ങൾ പൂർണ്ണ ലിസ്റ്റിന് man gimp ഉപയോഗിക്കാൻ കഴിയും.

കമാൻഡ് ലൈൻ വാദങ്ങൾ കമാൻഡ് ലൈൻ നിങ്ങൾ gimp-2.8 [OPTION...] [FILE|URI...] ആയി GIMP ആരംഭിക്കാൻ ഉപയോഗിക്കുന്ന ആയിരിക്കണം.

-?, --help

എല്ലാ commandline ഓപ്ഷനുകൾ ഒരു ലിസ്റ്റ് പ്രദർശിപ്പിക്കുന്നു.

--help-all

എല്ലാ സഹായ ഓപ്ഷനുകൾ കാണിക്കുക.

--help-gtk

കാണിക്കുക GTK + ഉപാധികൾ.

-v, --version

GIMP പതിപ്പും എക്സിറ്റ് അച്ചടിക്കുക.

--license

ലൈസൻസ് വിവരങ്ങളും എക്സിറ്റ് കാണിക്കുക.

--verbose

വിശദമായ തുടക്കത്തിൽ സന്ദേശങ്ങൾ കാണിക്കുക.

-n, --new-instance

ഒരു പുതിയ GIMP ഇൻസ്റ്റൻസ് ആരംഭിക്കുക.

-a, --as-new

പുതിയ ആയി തുറക്കുക ചിത്രങ്ങൾ.

-i, --no-interface

ഒരു യൂസർ ഇന്റർഫേസ് ഇല്ലാതെ പ്രവർത്തിപ്പിക്കുക.

-d, --no-data

പാറ്റേണുകൾ, ഘനമായ, പാലറ്റുകളോ, അല്ലെങ്കിൽ നയപ്രഖ്യാപന ലോഡ് ചെയ്യാതിരിക്കുക. ആരംഭിക്കൽ സമയവും ചെറുതാക്കിയ യോഗ്യൻ എവിടെ നോൺ ഇന്ററാക്ടീവ് സാഹചര്യങ്ങളിൽ പലപ്പോഴും ഉപയോഗപ്രദമായ.

-f, --no-fonts

ഏതെങ്കിലും ഫോണ്ടുകൾ ലോഡ് ചെയ്യാതിരിക്കുക. ഈ ഫോണ്ടുകൾ ഉപയോഗിക്കാൻ പാടില്ല, അല്ലെങ്കിൽ GIMP തളർന്ന കേടായതാണ് ഫോണ്ടുകൾ ബന്ധപ്പെട്ട പ്രശ്നങ്ങൾ കണ്ടെത്താൻ സ്ക്രിപ്റ്റുകൾ അതിവേഗം GIMP ലോഡ് സഹായിക്കുന്നു.

-s, --no-splash

ആരംഭിക്കുമ്പോൾ സ്പ്ലാഷ് സ്ക്രീൻ കാണിക്കരുത്.

--no-shm

GIMP പ്ലഗ്ഇന്നുകളും തമ്മിൽ പങ്കിട്ട മെമ്മറി ഉപയോഗിക്കരുത്.

--no-cpu-accel

പ്രത്യേക സിപിയു ആക്സിലറേഷൻ ഫംഗ്ഷനുകളും ഉപയോഗിക്കരുത്. കണ്ടെത്തൽ അല്ലെങ്കിൽ പവറ് ത്വരിതപ്പെടുത്തിയ ഹാർഡ്വെയർ അല്ലെങ്കിൽ പ്രവർത്തനങ്ങൾ പ്രവർത്തനരഹിതമാക്കുന്നതിനുള്ള പ്രയോജനകരം.

--session=name

ഈ GIMP സെഷനിൽ ഒരു വ്യത്യസ്ത sessionrc ഉപയോഗിക്കുക. തന്നിരിക്കുന്ന സെഷൻ പേര് സ്ഥിര sessionrc ഫയൽനാമത്തിലേക്ക് പ്പെടുന്നു.

--gimprc=filename

പകരം സ്ഥിര ഒറ്റ ബദൽ gimprc ഉപയോഗിക്കുക. gimprc ഫയൽ നിങ്ങളുടെ മുൻഗണനകൾ ഒരു റെക്കോർഡ് അടങ്ങിയിരിക്കുന്നു. പ്ലഗിനുകൾ പാതകൾ, മെഷീൻ നവ്യ വിവിധ ആയിരിക്കാം സാഹചര്യങ്ങളിൽ പ്രയോജനകരം.

--system-gimprc=filename

ഒരു ഇതര സിസ്റ്റം gimprc ഫയൽ ഉപയോഗിക്കുക.

-b, --batch=commands

നോൺ-ഇന്ററാക്ടീവ് കമാൻഡുകളുടെ സെറ്റ് നടപ്പിലാക്കുക. കമാൻഡുകളുടെ സെറ്റ് GIMP സ്ക്രിപ്റ്റിംഗ് വിപുലീകരണങ്ങൾ ഒന്ന് നിർവ്വഹിക്കാനാകും ഒരു സ്ക്രിപ്റ്റ് രൂപത്തിൽ ലോഹം ആണ്. കമാൻഡ് - എപ്പോഴാണ്, കമാൻഡുകൾ സ്റ്റാൻഡേർഡ് ഇൻപുട്ട് നിന്ന് വായിക്കാം.

--batch-interpreter=proc

ബാച്ച് കമാൻഡുകൾ പ്രോസസ്സ് ഉപയോഗിക്കാൻ നടപടിക്രമം വ്യക്തമാക്കുക. സ്ഥിര നടപടിക്രമം സ്ക്രിപ്റ്റ്-ഫു ആണ്.

--console-messages

പിശകുകൾ അല്ലെങ്കിൽ മുന്നറിയിപ്പുകൾ ന് ഡയലോഗ് പെട്ടികൾ popup ചെയ്യരുത്. പകരം കൺസോളിൽ സന്ദേശങ്ങൾ അച്ചടിക്കുക.

--pdb-compat-mode=mode

പലവക അനുയോജ്യത മോഡ് (ഓഫ് | ന് | താക്കീത്).

--stack-trace-mode=mode

ഒരു ക്രാഷ് കാര്യത്തിൽ ഡീബഗ് (| അന്വേഷണ | ഒരിക്കലും എല്ലായ്പ്പോഴും).

--debug-handlers
